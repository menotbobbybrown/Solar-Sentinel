Solar-Sentinel-AIO Version Manifest
Phase 5: EVA Energy Map (Gemini API Transition)
===============================================

# ============================================================================
# PHASE 5 OVERVIEW
# ============================================================================

This release transitions from Ollama (local LLM) to Gemini API and introduces
the EVA (Energy Value Architecture) system for intelligent per-device 
load management.

# ============================================================================
# CORE CHANGES
# ============================================================================

## AI/LLM
- Removed: Ollama local LLM service
- Added: Gemini API with google-generativeai==0.4.1
- Hermes Agent: Gemini 1.5 Pro with 12 Native Tools
- Function Calling for real-time system control

## EVA Features
- Energy Value Architecture (EVA) Registry
- Device Classification: CRITICAL, SHIFTABLE, LUXURY, PHANTOM
- Phantom Load Detection & Auto-Cut
- Optimal Energy Window Finder (4-hour sliding window)
- Pattern Learning Engine (InfluxDB queries)
- Smart Lockout (SHIFTABLE devices scheduled, not locked)
- Real-time Energy Map (MQTT/Node-RED/Grafana)

# ============================================================================
# HERMES AGENT - 12 TOOLS
# ============================================================================

1. get_system_status    - Check SOC, PV power, guard tier

3. unlock_appliance     - Unlock individual device

# ============================================================================
# EVA DEVICE PRIORITIES
# ============================================================================

CRITICAL  - Essential devices (refrigerator), never locked
SHIFTABLE - Can be scheduled for optimal windows (washer, EV)
LUXURY    - Comfort devices, locked in WARNING (AC, water heater)
PHANTOM   - Vampire loads, auto-cut when OFF but drawing power
